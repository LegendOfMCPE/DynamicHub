DynamicHub

DynamicHub provides the ability to code minigame plugins that can be adapted on any types of servers, hub servers or server networks with optimum performance and compatibility

What does DynamicHub do?

DynamicHub is a @PocketMine plugin. It provides a framework where continuous-game plugins (e.g. Kit PvP, Parkour) and match-based-game plugins (e.g. Spleef, SkyWars) can be easily built upon. DynamicHub provides the option to make a single-game server, a hub-based server (where you have multiple games connected with a hub world) or a network server (where player data are connected together).

DynamicHub offers the option to choose between file-based data saving and MySQL-based data saving. If you use MySQL-based data saving, you can share the data with other servers so that player data, e.g. inventories, are the same on all servers that have the same game type (a.k.a. module, which may refer to hub or a game) (they will get the same inventory loaded in any servers anytime as long as they are in the same module).

DynamicHub provides compatibility with SimpleAuth by the @PocketMine Team and ServerAuth by @EvolSoft, or you can choose not to use any auth plugins (although that is strongly discouraged). An extensive API

DynamicHub has an extensive API that allows games to be easily created. A developer can easily create a game plugin only thinking about what is happening inside the game, and DynamicHub will handle the rest of plugin compatibility.
